# AI海龟汤游戏需求研究

## 目标
做一个AI海龟汤游戏网站，让玩家可以和AI对话玩海龟汤。

## 调研发现
- 海龟汤游戏分为汤面（故事）和汤底（真相）
- 玩家通过提问来推理真相
- 主持人只回答"是"、"否"、"无关"

## 产品定位（确认）
- 玩法取向：娱乐社交（轻松、节奏快、低挫败）
- 对局形态：单人对AI
- 汤底查看：渐进解锁（用提示/进度逐步揭示关键点→因果→完整汤底）
- 内容边界：不做限制（但需要风控与风险提示机制，避免敏感内容带来的合规/体验问题）

## 目标用户与使用场景
- 新手玩家：想“随便玩一局”，希望有引导、能推进、别卡死
- 轻度推理爱好者：喜欢反转与脑洞，愿意多问几轮
- 场景：碎片化时间（3-10分钟/局）、睡前/通勤、朋友聊天时当话题引子

## 核心体验（玩家视角闭环）
1. 一键开局拿到汤面（短、好理解、留悬念）
2. 输入问题→AI用“是/否/无关/需更具体/无法判断”回答
3. 系统自动板书：沉淀“已确认事实/排除项/可疑方向”，减少重复提问
4. 卡住时用“提示/渐进看汤底”推进，持续获得反馈与进展感
5. 玩家提交“我的还原”：AI判定接近度并补全讲述
6. 结算复盘：公布完整汤底、关键转折问题、玩家提问高光

## 核心需求（MVP）
1. 生成与选择题目
   - AI生成汤面与汤底（同局必须一致）
   - 提供“换一题/重开同题”入口
2. 问答对局
   - 玩家输入问题
   - AI基于汤底判定回答（默认短答，必要时引导玩家把问题问具体）
   - 处理无法回答/多条件/过宽问题
3. 对话与板书
   - 记录对话历史（可折叠、可复制）
   - 自动总结已确认事实与排除项（板书区）
4. 渐进解锁汤底（关键功能）
   - 解锁层级建议：关键点列表 → 单个关键点 → 关键因果连接 → 完整汤底
   - 解锁成本建议：提示次数/进度条/轻量计分（娱乐向，避免“看了就没意思”）
5. 结算与复盘
   - 提交还原（玩家自由文本）
   - AI返回：接近度、缺失关键点提示、最终汤底讲述

## 非核心但强烈建议（提升娱乐感）
- 语气风格开关：严肃/幽默（默认幽默但不冒犯）
- 推荐提问：给2-3个“高信息增益”问题模板（避免卡住）
- 进度可视化：已解锁关键点 x/5、已用提示 y/3
- 高光时刻：本局“最佳问题/最离谱问题”自动评选

## 规则口径（AI裁判一致性）
- 允许的回答集（建议固定按钮/枚举）
  - 是 / 否 / 无关
  - 无法判断（汤底未定义或存在多解）
  - 需要更具体（问题过宽或含多条件）
- 判定原则
  - AI只允许基于“本局汤底”回答，不得临时追加设定
  - 若玩家问题可解释为多个含义，优先要求“更具体”，避免误导
  - 娱乐向允许轻引导，但不得改变事实口径

## 渐进解锁汤底（详细规则建议）
- 解锁触发
  - 玩家主动点击“查看线索/汤底”
  - 或卡住检测触发（连续多次无关/无法判断后建议使用提示）
- 解锁内容（示例）
  - Level 1：展示5-8个关键点标题（不解释因果）
  - Level 2：展示其中1个关键点完整句
  - Level 3：展示两点之间的一条因果连接（1-2句话）
  - Level 4：完整汤底（结算页优先展示）
- 风险点
  - 防止“前端直接拿到完整汤底”导致一键剧透：汤底应按层级按需下发

## 内容与风控（内容不限制但要可控）
- 风险点：自残、暴力、性、歧视、未成年人等敏感主题可能带来合规与体验问题
- 建议机制
  - 内容预警：题目生成后自动标记标签与风险级别，展示给玩家是否继续
  - 一键换题：不需要解释原因
  - 生成约束：默认避免极端/露骨细节，用“抽象表达”保留谜题性

## 关键数据（建议的最小数据模型）
- 汤题（Puzzle）
  - id、标题/主题、难度、标签、创建时间
  - 汤面（public）
  - 汤底（private，结构化：关键点列表、时间线、人物/地点/物件、不可变事实）
- 对局（Session）
  - sessionId、puzzleId、开始/结束时间
  - 对话消息列表（role、content、time、判定类型）
  - 板书快照（已确认事实/排除项/待验证假设）
  - 解锁进度（level、消耗次数）

## 非功能性需求（工程侧）
- 一致性：同局答案必须稳定（避免同问不同答）
- 性能：首轮生成与每次回答需要可接受延迟；支持缓存与重试
- 稳定性：网络/模型失败时给出可读错误与重试，不丢失对话
- 安全：不在日志/前端暴露完整汤底；对输入做基本过滤；限制超长输入与频率
- 观测：记录关键指标（开局、提问数、卡住率、提示使用率、通关率、退出点）

## 待确认问题（后续可补齐）
- 是否需要登录/保存战绩与题库收藏
- 题目来源：纯AI生成 vs 内置精选题库 + AI改写去重
- 难度分级标准与提示成本（次数/积分/冷却）
- UI形态：移动端优先还是桌面优先
