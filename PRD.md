# AI海龟汤游戏 PRD

## 1. 产品概述

### 产品是什么

一个AI驱动的海龟汤（情境猜谜/水平思考游戏）网站。玩家与AI主持人对话，通过封闭提问逐步推理出故事真相（汤底）。

### 核心体验一句话

玩家拿到一个"汤面"（一段离奇结果）→ 通过问"是/否/无关"的问题获取线索 → 用提示/渐进解锁推进卡点 → 最终还原真相并复盘。

### 产品定位

- **玩法取向**：娱乐社交（轻松、节奏快、低挫败）
- **对局形态**：单人对AI
- **汤底查看**：渐进解锁（免费无限，按层级逐步揭示关键点→因果→完整汤底）
- **内容边界**：AI生成题库（不限制主题，但需内容预警与一键换题机制）
- **用户规模预期**：碎片化时间用户，单局3-10分钟

## 3. 核心功能

### 3.1 用户系统（MVP必须做）

#### 注册与登录

- 支持邮箱/手机号注册，支持第三方登录（微信/GitHub等）
- 登录后保存：游戏历史、收藏题目、累计数据

#### 用户数据

- 累计游玩局数、解题成功率、平均提问轮次
- 收藏的题目
- 个人复盘记录（可选）

### 3.2 游戏大厅

#### 题目列表

- AI实时生成题目卡片：显示标题、难度（简单/中等/困难）、标签（日常/反转/恐怖/搞笑等）、内容预警标签（如有）
- 筛选与随机匹配：按标签/难度筛选，或"随机开始"一局
- 收藏功能：登录后可收藏题目

#### 开始游戏

- 点击卡片或"随机开始"进入对局
- AI同时生成"汤面"（对外展示）和"汤底"（结构化存储，裁判用）

### 3.3 游戏页面（核心体验）

#### 汤面展示

- 页面顶部展示"汤面"（一段离奇叙述）
- 可重复阅读，可复制分享

#### 聊天问答区

- 玩家输入问题，AI按以下规则回答：
  - **是**：该条件在汤底中成立
  - **否**：该条件在汤底中不成立
  - **无关**：该条件与汤底关键因果无关
  - **需要更具体**：问题过宽或含多个条件，需拆分
  - **无法判断**：汤底未定义或存在多解
- 娱乐向允许AI有轻引导语气（如"这个问题很有意思，但你要更具体一点哦"），但不改变事实口径
- 对话历史完整记录，可折叠/复制

#### 自动板书区

- 实时沉淀"已确认事实/排除项/可疑方向"
- 玩家可一目了然看到当前推理进度，减少重复提问
- 格式示例：
  - ✅ 已确认：他在室内 / 认识凶手
  - ❌ 排除：非自杀 / 非意外
  - 🔍 可疑：当晚有第三者在场？

#### 渐进解锁汤底（关键功能，免费无限）

- 玩家可主动点击"查看线索/汤底"，或卡住时系统自动建议
- 解锁层级（按需触发，后端按层级下发，不一次性暴露完整汤底）：
  - **Level 1 - 关键点列表**：展示5-8个关键点标题（无因果说明）
    > 例如：① 他以为的事 ② 真相是什么 ③ ...
  - **Level 2 - 单点揭示**：展示其中1个关键点的完整内容
  - **Level 3 - 因果连接**：展示两点之间的1条因果链（1-2句话）
  - **Level 4 - 完整汤底**：在结算页展示完整真相
- 每层解锁记录在Session中，防止前端一次性拉取

#### 提交还原

- 玩家可随时提交"我认为真相是……"（自由文本）
- AI判定接近度：
  - 覆盖主要关键点 → 结算页展示完整汤底 + 复盘
  - 部分正确 → 提示"还缺XX方向"
  - 完全不对 → "不太对哦，再想想？"

#### 结束游戏

- 玩家主动结束、或超时（默认30分钟）、或主动使用"查看完整汤底"
- 结束前AI自动复盘：本局关键转折问题、最佳问题、玩家推理路径

### 3.4 结算与复盘页

#### 内容

- 完整汤底（戏剧化讲述版）
- 玩家推理路径还原（关键问题时间线）
- 本局数据：提问轮次、提示使用次数、是否通关
- 社交分享：可生成分享图/文字（不含剧透）

#### 操作

- "再来一局"（同题或随机新题）
- "返回大厅"

### 3.5 内容预警与风控

#### AI生成控制

- 生成约束：避免极端/露骨细节，用"抽象表达"保留谜题性
- 题目生成后自动评估风险级别

#### 玩家侧保护

- 内容预警：进入高风险题目时弹窗提醒（如恐怖/暴力/社会敏感）
- 一键换题：随时可换，不需要理由
- 退出无成本：游戏中途可随时退出，不记录失败

## 5. 界面设计

### 整体风格

- **视觉基调**：神秘悬疑，深蓝色调（#0D1B2A / #1B263B），配合暖色高亮（#E0A458）
- **氛围**：夜间探索感，契合"挖掘真相"的叙事主题

### 页面布局

#### 游戏大厅

- 移动端优先，卡片式瀑布流/列表
- 每个卡片：标题 + 难度徽章 + 标签 + 内容预警角标（如有）
- 底部导航：首页/收藏/我的

#### 游戏页面

- 顶部：汤面卡片（固定，可展开/收起）
- 中部：聊天记录流（玩家问/AI答，左右区分）
- 右侧/底部：板书折叠面板（可展开看进度）
- 底部输入区：文本框 + 发送按钮 + "查看线索"快捷入口
- AI回答区：简短回答 + 判定标签（是/否/无关）

#### 结算页

- 仪式感揭晓：渐显动画展示完整汤底
- 复盘时间线：本局关键问答节点
- 操作按钮：再来一局 / 返回大厅

### 响应式策略

- 移动端优先，适配手机竖屏
- 平板/桌面端：板书区常驻右侧，聊天区更宽
- 不为桌面端牺牲移动端体验

## 6. 技术架构

### 技术栈

- **前端**：React + TypeScript + Tailwind CSS + Shadcn UI
- **后端**：Node.js + Express（REST API）
- **AI**：DeepSeek API（裁判问答 + 题目生成）
- **部署**：Vercel（前端 + Serverless API）
- **数据库**：Vercel Postgres（PostgreSQL）

### 部署与环境策略

- 运行形态：前端静态资源部署在Vercel，后端API以Vercel Serverless Functions运行
- 数据层：Vercel Postgres作为主库，连接串通过Vercel环境变量注入
- 环境划分：`development` / `preview` / `production` 三套环境隔离
- 密钥管理：AI API Key、JWT Secret、数据库连接串仅存在服务端环境变量，不下发到前端
- 发布策略：通过Git分支触发Preview环境，主分支合并后发布Production
  
### 核心数据结构

#### 题目 Puzzle

```

  keyPoints: string[]                   // 关键点列表（5-8条）
  timeline: string[]                    // 时间线/因果链

```

#### 对局 Session

  answerType?: '是' | '否' | '无关' | '需要更具体' | '无法判断'
  timestamp: Date

  confirmed: string[]      // 已确认事实
  excluded: string[]      // 排除项
  suspects: string[]      // 可疑方向

  totalGames: number

### API 设计（核心接口）

#### 题目

- `POST /api/puzzles` - AI生成新题目
- `GET /api/puzzles/random` - 随机获取一道题
- `GET /api/puzzles/:id` - 获取题目详情（不含汤底）
- `POST /api/puzzles/:id/favorite` - 收藏题目

#### 对局

- `POST /api/sessions` - 创建新对局（携带puzzleId）
- `GET /api/sessions/:id` - 获取对局详情（含当前板书/解锁层级）
- `POST /api/sessions/:id/messages` - 发送玩家消息，AI回复（包含判定）
- `POST /api/sessions/:id/unlock` - 触发一次汤底解锁（按层级）
- `POST /api/sessions/:id/submit` - 提交还原，AI判定
- `POST /api/sessions/:id/end` - 结束对局

#### 用户

- `POST /api/auth/register` - 注册
- `POST /api/auth/login` - 登录
- `POST /api/auth/refresh` - 刷新访问令牌
- `POST /api/auth/logout` - 退出登录（使刷新令牌失效）
- `GET /api/users/me` - 获取当前用户信息与统计
- `GET /api/users/me/favorites` - 获取收藏列表

### 鉴权与会话策略

- 鉴权方式：JWT Access Token + Refresh Token
- Access Token：短时效（建议15-30分钟），用于API鉴权
- Refresh Token：长时效（建议7-30天），仅用于换取新Access Token
- 令牌存储：前端优先使用HttpOnly Cookie；若使用本地存储需额外防XSS策略
- 退出登录：服务端将Refresh Token加入失效列表，禁止继续刷新
- 受保护接口：收藏、用户信息、历史记录、提交结果写入等需登录后访问

### AI 裁判一致性保障

- 汤底结构化存储：所有回答严格基于`bottom`字段，不临时追加
- Session状态记录：每轮回答后更新板书，防止不同轮次矛盾
- 解锁按层级下发：后端控制，前端不存储完整汤底直到结算
- 同问同答：对同一session中语义相近问题进行归一化校验，判定冲突时优先返回历史口径
- 可追溯性：保存每轮判定类型与依据摘要，便于线上问题排查

### 性能与稳定性

- 题目生成结果缓存，避免重复调用AI
- 对话上下文压缩：超出阈值时自动摘要，节省token
- API限流：单用户每分钟最多20条消息；未登录IP每分钟最多10条消息
- 模型失败时返回友好错误，引导重试，不丢失对话状态
- 失败降级：当AI接口连续失败时，启用兜底文案并建议稍后重试或换题
- 重试策略：对可重试错误最多重试2次，超过后返回标准错误码

### 安全

- 汤底不在日志中明文记录完整因果（只记录关键点标题）
- 前端不存储完整汤底（Session中不解锁到Level 4不透出）
- 输入过滤：防注入、超长文本截断
- AI内容过滤：对生成结果做敏感词检测，命中则替换或拒绝
- 防滥用：`/messages` 与 `/unlock` 接口增加频率限制与行为审计

### API 规范与错误码

- 响应结构统一

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "req_xxx"
}
```

- 常见错误码
  - `40001` 参数错误
  - `40101` 未登录或令牌无效
  - `40301` 无权限访问
  - `40401` 资源不存在
  - `40901` 状态冲突（如重复结束对局）
  - `42901` 请求过于频繁
  - `50001` 服务内部错误
  - `50301` AI服务暂不可用

### 幂等与并发处理

- `POST /api/sessions/:id/unlock`：同一层级重复请求返回同一结果，不重复扣减状态
- `POST /api/sessions/:id/end`：重复调用应返回当前终态，不重复结算
- `POST /api/sessions/:id/submit`：同一请求可用`clientRequestId`防重
- 并发写保护：基于`session.version`或事务锁，避免板书与解锁状态错乱

## 9. P0验收标准（可测试）

### 9.1 用户注册/登录

- 可使用邮箱注册并登录成功，成功率≥99%
- 登录后访问`/api/users/me`返回用户信息；未登录访问返回`40101`
- Access Token过期后可通过`/api/auth/refresh`刷新成功

### 9.2 AI生成题目 + 大厅列表

- 点击随机开始后5秒内返回汤面（P90 < 5秒）
- 题目对象包含`surface`且不包含完整汤底明文
- 内容预警字段可用（高风险题材可标注）

### 9.3 核心问答对局

- 玩家提问后返回判定类型（是/否/无关/需要更具体/无法判断）
- 同一session内语义等价提问判定一致率≥99%
- 失败重试后仍失败时返回`50301`，并给出可读提示

### 9.4 渐进解锁汤底

- Level 1-4可逐级解锁，不能跨级直接获取完整汤底
- 前端网络面板中在Level 4前不得出现完整汤底字段
- 重复点击同一解锁层级不造成状态异常（幂等）

### 9.5 结算复盘页

- 提交还原后返回接近度结果与缺失方向提示
- 结算页展示完整汤底、提问轮次、提示使用次数
- 再来一局可在3秒内创建新session并进入新对局

### 9.6 指标口径定义

- 通关率：`completed且判定通过的局数 / 已结束局数`
- 平均提问轮次：`已结束局总提问数 / 已结束局数`
- 卡住率：出现连续3次`无关或无法判断`的局数占比
